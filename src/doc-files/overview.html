JJCron - Just a Java Cron is cron-like program written in Java which can be used to periodically execute given tasks. JJCron is standalone application with cron function but can be also used as library for executing tasks in other programs. This can be done through modular and lightweight desing of internal architecture.

Whole application was developed using Netbeans IDE with Java 8 language standard. Under lesser versions program do not have to compile properly. Due to Java usage whole application is multiplatform and can be used on whatever system Java can run.

JJCron provides basic cron functionality without some special features concerning crontab configuration. But in addition some functionality is added. For instance there is seconds time unit column in crontab for even more delicate scheduling. Another improvement is that Java classes can be defined in crontab loaded and scheduled just as normal cron tasks. This is achieved by putting name of class between class XML tags.

Crontab

Scheduled tasks are handed over to JJCron with crontab configuration file. Format of this configurations is standard and very similar to actual crontab used in proper cron implementations.

Example of crontab format:

 #  ┌──────────── sec (0 - 59)
 #  │ ┌───────────── min (0 - 59)
 #  │ │ ┌────────────── hour (0 - 23)
 #  │ │ │ ┌─────────────── day of month (1 - 31)
 #  │ │ │ │ ┌──────────────── month (1 - 12)
 #  │ │ │ │ │ ┌───────────────── day of week (1 - 7) (1 = Monday, 7 = Sunday)
 #  │ │ │ │ │ │
 #  │ │ │ │ │ │
 #  * * * * * *  command to execute

Main features

  • All code written in pure Java using cutting edge Java 8 version
  • Thanks to Java JJCron is multiplatform
  • Modular and lightweight desing
  • Easy to understand and use
  • Java classes can be used as tasks and can be scheduled to execution
  • Will be further improved to support RMI remote management

Drawbacks

  • Not all features from crontab standard format are implemented
  • Day of week time unit is a bit non-standard with its ordering
  • Due to Java restrictions only single program can be executed (that means no pipes, redirections or program chaining)
  • Test are not written yet!
